# CS-320-Software-Test-Automation
Portfolio for CS-320 Software Automation

How can I ensure that my code, program, or software is functional and secure?
I ensure my software is functional and secure by writing unit tests that validate expected behavior and edge cases before deployment. Using JUnit allowed me to verify that methods returned correct results under normal and invalid inputs. I also focused on input validation, clear method contracts, and defensive programming to reduce the risk of runtime errors and security issues. Consistent testing helps catch defects early and ensures the software behaves as intended.

How do I interpret user needs and incorporate them into a program?
I interpret user needs by translating requirements into clear, testable behaviors within the code. User expectations guide how methods are designed, what inputs are allowed, and what outputs are returned. By aligning tests directly with requirements, I can confirm that the program meets user needs and handles real-world scenarios reliably. This approach ensures functionality is driven by requirements rather than assumptions.

How do I approach designing software?
I approach software design by focusing on modularity, readability, and testability. Breaking the program into smaller, well-defined components makes the code easier to maintain and test. Writing tests alongside development helps guide design decisions and ensures each component functions independently. This structured approach results in software that is easier to extend, debug, and validate.
